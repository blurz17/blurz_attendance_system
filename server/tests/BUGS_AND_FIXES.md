### BUG-1 & BUG-2 — `verify_qr_token` timezone & exception swallowing

**File:** `core/security.py`

**Problem:**
```python
# CURRENT — BROKEN
try:
    ...
    expires_at = datetime.fromisoformat(payload["expires_at"])
    if datetime.now(timezone.utc).replace(tzinfo=None) > expires_at:
        from core.errors import QRCodeExpired
        raise QRCodeExpired()          # ← SWALLOWED by the except below!
    return payload
except (ValueError, KeyError, json.JSONDecodeError):
    raise InvalidToken()              # ← catches QRCodeExpired too!
```

**Fix:**
```python
def verify_qr_token(token: str) -> dict:
    try:
        parts = token.split(".", 1)
        if len(parts) != 2:
            raise InvalidToken()

        encoded_payload, received_sig = parts
        payload_str = base64.urlsafe_b64decode(encoded_payload.encode()).decode()

        expected_sig = hmac.new(
            config.HMAC_SECRET.encode(),
            payload_str.encode(),
            hashlib.sha256,
        ).hexdigest()

        if not hmac.compare_digest(received_sig, expected_sig):
            raise InvalidToken()

        payload = json.loads(payload_str)

    except (ValueError, KeyError, json.JSONDecodeError):
        raise InvalidToken()

    # ✅ Check expiry OUTSIDE the except block so QRCodeExpired propagates
    expires_at_str = payload.get("expires_at")
    if not expires_at_str:
        raise InvalidToken()

    try:
        expires_at = datetime.fromisoformat(expires_at_str)
    except ValueError:
        raise InvalidToken()

    # ✅ Compare timezone-aware datetimes consistently
    now = datetime.now(timezone.utc)
    if expires_at.tzinfo is None:
        expires_at = expires_at.replace(tzinfo=timezone.utc)

    if now > expires_at:
        raise QRCodeExpired()

    return payload
```

link = f"{config.frontend_url}/activate/{token}"

### BUG-4 — Quiz router: route ordering shadows `/instructor/submissions/{id}`

**File:** `core/client/quiz/router.py`

**Problem:**
```python
# CURRENT ORDER — /{quiz_id} matches "instructor" first!
@quiz_router.get("/{quiz_id}", ...)           # line ~40
async def get_quiz_details(...): ...

@quiz_router.get("/instructor/submissions/{quiz_id}", ...)   # too late!
async def get_quiz_submissions(...): ...
```

**Fix:** Move static-path routes BEFORE parameterised ones:
```python
# ✅ CORRECT ORDER
@quiz_router.get("/available", ...)
@quiz_router.get("/instructor", ...)
@quiz_router.get("/instructor/submissions/{quiz_id}", ...)  # static before param
@quiz_router.post("/create", ...)
@quiz_router.post("/{quiz_id}/submit", ...)
@quiz_router.get("/{quiz_id}", ...)                         # param routes LAST
```

### BUG-6 — Quiz with no correct answer accepted silently

**File:** `core/client/quiz/service.py`

**Problem:** `create_quiz` does not validate that each question has at least one correct choice.

**Fix:** Add validation before persisting questions:
```python
for idx, q_data in enumerate(questions_data):
    correct_count = sum(1 for c in q_data.choices if c.is_correct)
    if correct_count == 0:
        raise HTTPException(
            status_code=422,
            detail=f"Question {idx + 1} has no correct answer marked."
        )
    if not q_data.choices:
        raise HTTPException(
            status_code=422,
            detail=f"Question {idx + 1} must have at least one choice."
        )
```

### BUG-7 — Double instructor verification in attendance report

**File:** `core/client/attendance/service.py`

**Problem:**
```python
async def get_course_attendance_report(...):
    await _verify_instructor(...)           # 1st check
    result = await get_full_course_attendance_report(...)  # calls _verify_instructor again
```

**Fix:** Either inline the query or add a `_skip_auth=False` flag:
```python
async def get_course_attendance_report(course_id, user_id, user_role, session):
    await _verify_instructor(user_id, course_id, user_role, session)
    # Reuse internal logic directly, skipping the second auth check
    return await _build_attendance_report(course_id, session)
```

### BUG-9 — Bulk upload session reuse after rollback

**File:** `core/admin/service.py`

**Problem:**
```python
for row in reader:
    try:
        await create_single_user(data, session)
        ...
    except Exception as e:
        await session.rollback()   # ← session is now in undefined state
        results.append(...)         # next iteration reuses broken session
```

**Fix:** Use a fresh nested savepoint per row:
```python
for row in reader:
    try:
        async with session.begin_nested():   # savepoint per row
            await create_single_user(data, session)
        results.append(BulkUploadRowResult(..., success=True))
    except Exception as e:
        # savepoint auto-rolled back; outer session is still valid
        results.append(BulkUploadRowResult(..., success=False, error=str(e)))
```

### BUG-10 — Fragmented is_active check

**File:** `core/dependencies.py`

**Problem:** `get_current_user` fetches the user without checking `is_active`.
`RoleChecker` checks it, but unauthenticated routes using `Depends(get_current_user)` alone (like `/me`) accept inactive users.

**Fix:** Check `is_active` in `get_current_user`:
```python
async def get_current_user(...) -> User:
    user = await get_user_by_email(token["user"]["email"], session)
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    if not user.is_active:
        raise AccountNotActive()
    return user
```
